


请遵守下面这些要求:

### 编程, 写代码要求
- 不要写 emoji
- 不要写大块的注释, 
- 尽量少写 try 语句，打印完整的报错信息
- 文档说明要简化，能写2句，不要写5句。
- 不要写太多总结文档。
- 写完代码之后，检查一下是否有语法错误，并处理。


### 文件操作
- 创建文件夹：先检查是否已存在，已存在就跳过。
- 注意我是windows 平台，文件路径要注意斜杠的问题。



### python 相关
- 新建 python 项目，一定要想新建一个 venv
- 经常检查并更新 requirement.txt 和 readme


### 我的 UI 喜好
- 强烈偏好暗色背景作为默认启动状态，深灰色就行。
- 删掉所有的鼠标悬浮动画，很低级
- 组件多使用：悬浮面板。
- 追求“高级感”与“克制”的配色  ，极度厌恶：俗套的、大面积的“科技蓝”、“深紫”渐变，认为这很“庸俗”。
- 导航栏的易用性与统一性 ：不喜欢分散的按钮布局。
- 按钮必须带有文字标签，不能仅有图标，强调“直观”和“简易使用”。


### 补充说明
- 涉及底层架构、API 平台或全局变量变更时，必须先进行全局搜索，严禁只修改配置文件。
- 长耗时任务必须提供进度反馈。
- 修改之前，做一个简单的影响分析
- 下载一些包，如果比较慢， 考虑换个镜像。
- 写完代码，增加功能，修改功能，更新 readme, 记录一下
